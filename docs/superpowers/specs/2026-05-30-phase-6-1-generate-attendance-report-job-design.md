# Phase 6.1 — GenerateAttendanceReport Job Design

## Context

Phase 6.1 implements the `GenerateAttendanceReport` queued job, which is already dispatched by `app/Filament/Admin/Pages/ReportPage.php` and `app/Filament/Faculty/Resources/AttendanceSessions/Tables/AttendanceSessionTable.php`. The job generates PDF, CSV, or XLSX attendance reports, stores them to disk, and notifies the requesting user when ready.

## Schema Change

`session_exports.session_id` must be made nullable. General reports (department, course, date range) are not tied to a single session. The existing `session-exports.download` route and `SessionExportResource` already handle `session_id = null` gracefully.

**Migration:** `make_session_id_nullable_on_session_exports`

```php
Schema::table('session_exports', function (Blueprint $table) {
    $table->foreignId('session_id')
        ->nullable()
        ->change();
});
```

## Job Constructor

Matches the signature already used by `ReportPage::generateReport()` and `AttendanceSessionTable`:

```php
public function __construct(
    public readonly string $type,       // department|course|faculty|student|date_range
    public readonly ?int $departmentId,
    public readonly ?int $courseId,
    public readonly ?string $from,
    public readonly ?string $to,
    public readonly string $format,     // pdf|csv|xlsx
    public readonly int $requestedBy,
) {}
```

## Job Lifecycle

1. **Start:** Create `SessionExport` with `status=processing`, `format`, `requested_by`, `session_id=null`
2. **Generate:** Build query → produce file → store to `storage/app/reports/{uuid}.{ext}` (private)
3. **Complete:** Update `SessionExport` to `status=ready`, set `file_path` and `expires_at = now()+24h`
4. **Notify:** `Notification::make()->title('Report ready')->body('...')->actions([...])->sendToDatabase($user)`
5. **failed():** Update `SessionExport` to `status=failed`, send failure notification

## Queue Configuration

```php
public int $tries = 3;
public array $backoff = [1, 5, 10];
```

## Report Data

All types query `attendance_records` with these columns:

| Column | Source |
|---|---|
| Student | `student → user → name` |
| Roll No | `student → roll_no` |
| Course | `session → course → code` |
| Session Date | `session → started_at` |
| Status | `status` |
| Marked At | `marked_at` |
| Risk Score | `risk_score` |

### Filter Logic Per Type

| Type | Primary filter | Date filter |
|---|---|---|
| `department` | `session → course → department_id = $departmentId` | `session → started_at` between `$from` and `$to` |
| `course` | `session → course_id = $courseId` | same |
| `faculty` | `session → faculty_id = $departmentId` (reuses field; UI will wire up properly in Phase 7+) | same |
| `student` | `student_id = $courseId` (reuses field; UI will wire up properly in Phase 7+) | same |
| `date_range` | none (date range only) | same |

If the required filter ID is absent, the job skips the primary filter and falls back to date range only, rather than failing.

## Export Classes

**CSV/XLSX:** `app/Exports/AttendanceReportExport.php`
- Implements `FromQuery`, `WithHeadings`
- Accepts a pre-built `Builder` instance
- Streams query directly — no full in-memory load

**PDF:** `resources/views/reports/attendance-report.blade.php`
- Header: report type, date range, generated timestamp
- Body: table of rows (same columns as above)
- Footer: "Generated by QR Attendance System"
- Rendered via `Pdf::loadView(...)->output()`, stored as string to disk

## Notification

Sent via `Filament\Notifications\Notification::make()` with `->sendToDatabase($user)` (persistent, appears in notification bell).

Success notification:
- Title: `"Report ready"`
- Body: `"Your attendance report is ready for download. The link expires in 24 hours."`
- Action: `Actions\Action::make('download')->url(URL::temporarySignedRoute('session-exports.download', now()->addHours(24), ['export' => $export->id]))`
- Colour: success

Failure notification:
- Title: `"Report failed"`
- Body: `"Your attendance report could not be generated. Please try again."`
- Colour: warning

## Tests (`tests/Feature/Jobs/GenerateAttendanceReportTest.php`)

| Test | Approach |
|---|---|
| `test_job_creates_session_export_record_on_completion()` | `Storage::fake()`, `Notification::fake()`, dispatch job, `assertDatabaseHas('session_exports', ['status' => 'ready'])` |
| `test_job_sets_status_to_ready_with_file_path()` | Assert `file_path` non-null, `expires_at` within 25 hours of now |
| `test_job_sets_status_to_failed_on_exception()` | Mock Excel to throw, call `failed(new \Exception)`, assert `status=failed` |
| `test_pdf_format_generates_pdf_file()` | Dispatch with `format=pdf`, assert `Storage::exists($export->file_path)` |
| `test_xlsx_format_generates_xlsx_file()` | Dispatch with `format=xlsx`, assert `Storage::exists($export->file_path)` |

`Storage::fake()` and `Notification::fake()` in `beforeEach`. Queue runs synchronously (`Queue::fake()` not used — job dispatched directly via `new GenerateAttendanceReport(...)->handle()`).
